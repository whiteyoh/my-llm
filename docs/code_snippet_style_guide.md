# Code Snippet Style Guide

Use these rules for all technical-manual snippets:

- Use **monospace** for code, commands, file paths, config values, and terminal output.
- Keep code font around **9-11 pt** for readable print/PDF output.
- Place snippets in clearly separated blocks with visible padding and spacing.
- Label snippets where helpful: `bash`, `python`, `yaml`, `json`, `sql`.
- Keep snippets short and focused; avoid full-file dumps unless needed.
- Show relevant lines only, but keep enough context to avoid ambiguity.
- Use consistent indentation and formatting.
- Separate commands from expected output in different blocks.
- Keep copy/paste safe: no smart quotes, hidden characters, or accidental `$` prompts.
- Explain what each snippet does before or after the block.
- Mark placeholders clearly, for example `<namespace>` or `<filename>`.
- Avoid screenshots of code unless visual layout is the actual point.
